# Команды Telegram-бота FanApp (ЧМ-2026)

Полный справочник команд бота. **Источник истины — [`bot/index.js`](bot/index.js)**; этот файл держим синхронным с ним.

- **Публичные** команды доступны всем пользователям.
- **Админские** команды работают только для `ADMIN_ID` (env-переменная); остальным бот молча не отвечает.
- Все команды пишутся боту в личку. `/score`, `/pen`, `/keeper` — **по одной команде на сообщение** (многострочный ввод НЕ поддерживают). Исключение — `/scorer`: принимает пакет из нескольких строк за раз.

---

## Публичные

| Команда | Что делает |
|---|---|
| `/start` | Приветствие + кнопка «Открыть приложение». Регистрирует пользователя (для рассылок и лидерборда). |

## Админские

### Статистика

| Команда | Что делает |
|---|---|
| `/stats` | Число подписчиков и число игроков в лидерборде. |
| `/help` | Краткая справка по админ-командам (в чате бота). |

### Результаты матчей (`/score`, `/pen`)

`/score` **идемпотентна и delta-безопасна**: повторный или корректирующий ввод НЕ задваивает очки (тот же счёт → лидерборд не меняется; другой счёт/итог → пересчёт разницы). Три формы:

| Форма | Пример | Что делает |
|---|---|---|
| Обычный / финальный счёт | `/score m01 2:1` | Фиксирует счёт матча (группа или нокаут, решённый в основное время). Показывает цифры в «оконченных» + сверяет очки лидерборда. |
| Серия пенальти | `/score m75 1:1 MAR` | Нокаут с серией. Последний токен — **кто ПРОШЁЛ**: `HOME` / `AWAY` или TLA сборной (`MAR`). Правило «прошёл ⇒ выиграл серию»: цифры серии знать не обязательно, достаточно победителя. Проставляет `winner`. |
| Доп. время | `/score m82 2:2 дв 3:2` | Матч, решённый в доп. время: 90′ = `2:2`, 120′ (итог) = `3:2`. Победитель — по счёту 120′. На главной покажет `(д.в.)`. Ключевые слова: `дв` / `д.в.` / `et` / `ет`. |
| Доп. время → серия | `/score m82 1:1 дв 1:1 MAR` | Если и в 120′ ничья — добавь прошедшего (трактуется как серия пенальти). |

| Команда | Что делает |
|---|---|
| `/pen m75 4:5` | Вручную вносит **верный счёт серии пенальти** (формат `хозяева:гости`). Нужна, т.к. football-data на сериях отдаёт недостоверный счёт (ничья). Счёт серии — источник истины: победитель = у кого больше → команда заодно проставляет `winner` и пересверяет очки. На главной покажет `(пен. 4:5)`. Матч должен быть уже зафиксирован (`/score`). Ничейный счёт серии отклоняется. |

> **Почему по пенальти отдельная логика.** football-data.org на матчах с серией часто присылает `winner: null` и ничейный счёт серии (5:5 / 1:1). Поэтому очки начисляются по тому, **кто прошёл** (`winner`), а не по цифрам серии. Если FD не дал `winner`, матч не фиксируется автоматически — до ручного `/score … <прошедший>` или `/pen`.

### Бомбардиры (`/scorer`)

Каждая строка сообщения — отдельная команда: можно **вставлять пакетом** (несколько строк за раз). После применения список пересортировывается по голам, затем ассистам.

| Команда | Что делает |
|---|---|
| `/scorer` | Выводит текущий список бомбардиров (группировка по сборным, реальные номера). При 100+ игроках бьётся на несколько сообщений со сквозными номерами. |
| `/scorer N голы ассисты [матчи]` | Обновляет игрока №N. |
| `/scorer add Имя TLA голы ассисты [матчи]` | Добавляет нового игрока (TLA — 3-буквенный код сборной). |
| `/scorer del N` | Удаляет игрока №N. |
| `/scorer ren N Новое Имя` | Переименовывает игрока №N (привязка `fdId` к API сохраняется). |
| `/scorer_reset` | Полностью очищает список бомбардиров. |

### Вратари (`/keeper`)

| Команда | Что делает |
|---|---|
| `/keeper` | Список вратарей с их статистикой. |
| `/keeper N сухие минуты [матчи]` | Обновляет вратаря №N (сухие матчи, минуты без пропущенного гола, сыгранные матчи). Список пересортировывается по сухим матчам → минутам. |

### Рассылки

| Команда | Что делает |
|---|---|
| `/send текст` | Рассылает текст всем подписчикам (Markdown) с кнопкой открыть приложение. |
| `/result 🇫🇷 Франция 2:1 🇧🇷 Бразилия` | Рассылает оформленное объявление результата матча с кнопкой «Смотреть таблицу». |
| `/photo url \| подпись` | Рассылает фото по URL с подписью и кнопкой открыть приложение. |
| _(любое сообщение)_ | **Любое НЕ-командное сообщение от админа** (текст / фото / видео) автоматически уходит рассылкой всем подписчикам. |

## Идентификаторы матчей

- Группа: `m01`…`m72`.
- Плей-офф: `m73`+ (R32 — `m73`…`m88`, R16 — `m89`…`m96`, 1/4 — `m97`…`m100`, и т.д.).
- Сопоставление TLA-пар с id — в [`bot/match-lookup.js`](bot/match-lookup.js) (`MATCH_LOOKUP`).

## Примечания

- Внесённый через `/score`/`/pen` результат мгновенно отражается в мини-аппе (в Telegram может понадобиться перезайти из-за кэша); на следующих циклах поллер подтверждает его из хранилища (источник истины).
- Правки нокаута (`winner` / 90′ / 120′) пересчитывают очки delta-безопасно — задвоения нет.
- Разовые корректировки исторических матчей делаются серверными миграциями в `bot/index.js` (гард-флаг в Redis, срабатывают один раз на деплое).
